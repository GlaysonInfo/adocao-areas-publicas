Problemas recorrentes e correções
A) PowerShell Invoke-RestMethod (IRM) com POST JSON

Sintomas:

FST_ERR_CTP_INVALID_CONTENT_LENGTH

Unexpected non-whitespace character after JSON...

Correção:

Usar curl.exe com --data-binary (corpo exato) via helper PostJson().

Helper recomendado:

$base = "http://127.0.0.1:3001"

function PostJson($url, $obj) {
  $json = ($obj | ConvertTo-Json -Depth 50 -Compress)
  $tmp = New-TemporaryFile
  [System.IO.File]::WriteAllText($tmp.FullName, $json, [System.Text.UTF8Encoding]::new($false))
  $out = curl.exe -sS -X POST $url -H "Content-Type: application/json" --data-binary "@$($tmp.FullName)"
  Remove-Item $tmp.FullName -Force
  if (-not $out) { return $null }
  return ($out | ConvertFrom-Json)
}

B) Prisma generate EPERM (Windows file lock)

Sintoma:

EPERM: operation not permitted, rename ... query_engine-windows.dll.node.tmp  ...

Correção:

Encerrar processos Node/tsx e remover .prisma:

Get-Process node -ErrorAction SilentlyContinue | Stop-Process -Force
Remove-Item -Recurse -Force node_modules\.prisma -ErrorAction SilentlyContinue
npx prisma generate

C) PostgreSQL (P1001 / P1000)

P1001: servidor não acessível → verificar serviço e porta 5432.

P1000: falha de autenticação → ajustar password do usuário no DATABASE_URL.
